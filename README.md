# 🚀 Automated GRE Tunnel & Port Forwarder




این اسکریپت به شما کمک می‌کند تا به سادگی و به صورت کاملاً خودکار، یک تانل امن **GRE** بین دو سرور (ایران و خارج) برقرار کرده و پورت‌های دلخواه خود را از سرور ایران به خارج فوروارد کنید. همچنین این اسکریپت دارای قابلیت **Systemd Service** جهت اجرای خودکار پس از ریبوت (Reboot) سرور و کنترل هوشمند گیت‌وی‌های خارج از رنج (`onlink`) است.

---

## ✨ ویژگی‌ها (Features)
* 🛠️ **کاملاً خودکار و هوشمند:** تشخیص خودکار کارت شبکه فعال و Gateway اصلی سرور.
* 🔄 **پشتیبانی از ریبوت (Persistence):** ساخت خودکار سرویس `systemd` برای بالا آمدن تانل بعد از روشن شدن سرور.
* 🛡️ **مکانیزم ضد تکرار:** بررسی تانل‌های قدیمی و پاک‌سازی آن‌ها قبل از اجرای مجدد برای جلوگیری از تداخل (Idempotent).
* 🎛️ **حالت دیباگ (Debug Mode):** امکان عیب‌یابی خط به خط با فلگ `--debug`.
* 🎨 **رابط کاربری فینگلیش و رنگی:** تعامل راحت در ترمینال‌های بدون فونت فارسی.

---

## 🗺️ ساختار تانل و نمونه آی‌پِی‌ها (Example Architecture)

در این راهنما و برای تست، از آی‌پِی‌های فرضی زیر استفاده شده است:
* **IP Server Iran (آی‌پِی ایران):** `192.168.10.50`
* **IP Server Kharej (آی‌پِی خارج):** `192.168.20.90`
* **GRE Subnet (رنج داخلی تانل):** `10.10.0.0/30`
  * آی‌پِی تانل خارج: `10.10.0.1`
  * آی‌پِی تانل ایران: `10.10.0.2`

## 🚀 نحوه راه‌اندازی (Setup Guide)

این مراحل را باید روی **هر دو سرور** (ابتدا خارج و سپس ایران، یا برعکس) اجرا کنید.

### ۱. دریافت و آماده‌سازی اسکریپت


### بصورت خودکار


```
curl -Lo gre_port_fw.sh https://raw.githubusercontent.com/ExtremeDot/eXtreme-GreTunnel-With-port-Forwarding/main/gre_port_fw.sh

chmod +x gre_port_fw.sh

sudo ./gre_port_fw.sh

```
### بصورت دستی
ابتدا وارد سرور خود شده و یک فایل برای اسکریپت بسازید:
```bash
nano gre_port_fw.sh
```

کد اسکریپت را داخل آن کپی کرده، فایل را ذخیره کنید (Ctrl+O و سپس Ctrl+X) و دسترسی اجرایی به آن بدهید:


[Link Script](https://github.com/ExtremeDot/eXtreme-GreTunnel-With-port-Forwarding/blob/main/gre_port_fw.sh)


```
chmod +x gre_port_fw.sh

```




----

### ۲. اجرای اسکریپت روی سرور خارج (Kharej)

اسکریپت را با دسترسی روت اجرا کنید:

```
sudo ./gre_port_fw.sh
```

### مراحل پرسش و پاسخ:

از شما آی‌پِی ایران را می‌خواهد -> 192.168.10.50 (آی‌پِی اصلی خود را بزنید)

از شما آی‌پِی خارج را می‌خواهد -> 192.168.20.90 (آی‌پِی اصلی خود را بزنید)

مقدار MTU پیش‌فرض را تایید کنید (اینتر بزنید تا روی 1340 بماند).

پورت‌ها را تایید کنید (اینتر بزنید یا پورت‌های خود را با , جدا کنید).

در بخش انتخاب سرور، عدد 2 (Server KHAREJ) را وارد کنید.

در نهایت برای سوال Systemd، کلید y را بزنید تا سرویس ابدی شود.

---

### ۳. اجرای اسکریپت روی سرور ایران (Iran)
مجدداً اسکریپت را روی سرور ایران اجرا کنید:


```
sudo ./gre_port_fw.sh
```

### مراحل پرسش و پاسخ:

آی‌پِی ایران را وارد کنید -> 192.168.10.50

آی‌پِی خارج را وارد کنید -> 192.168.20.90

مقدار MTU را تایید کنید (1340).

پورت‌هایی که قرار است کاربران به آن وصل شوند را وارد کنید (مثلاً: 80,443,2053).

در بخش انتخاب سرور، عدد 1 (Server IRAN) را وارد کنید.

در نهایت برای سوال Systemd

، کلید y را بزنید تا قوانین فوروارد پورت پس از ریبوت هم پایدار بمانند.

### 🔍 دستورات مدیریت و عیب‌یابی (Management & Debugging)
بررسی وضعیت سرویس (Systemd Status)
برای اینکه ببینید سرویس به درستی فعال است یا خیر:

```
sudo systemctl status gre-tunnel.service
```
ریستارت کردن تانل و قوانین فایروال
اگر تغییراتی دادید یا خواستید سرویس را مجدد راه‌اندازی کنید:

```
sudo systemctl restart gre-tunnel.service
```
بررسی کارت شبکه تانل
برای اطمینان از بالا بودن اینترفیس gre1 و داشتن آی‌پِی داخلی:

```
ip a show gre1
```

بررسی قوانین فوروارد پورت (روی سرور ایران)
برای دیدن اینکه پورت‌ها به درستی در IPTABLES هدایت می‌شوند:

```
sudo iptables -t nat -L PREROUTING -n -v
```

اجرای اسکریپت در حالت دیباگ (Debug Mode)
اگر در اجرای اسکریپت به مشکلی خوردید و خواستید خط به خط دستورات اجرایی لینوکس را ببینید، از فلگ زیر استفاده کنید:

```
sudo ./gre_port_fw.sh --debug
```

💡 نکته: این اسکریپت مشکل قطع شدن اینترنت سرورهایی که گیت‌وی آن‌ها خارج از ساب‌نت اصلی است را با استفاده از تگ onlink در مسیردهی روت‌ها به طور کامل حل می‌کند.
